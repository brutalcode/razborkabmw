Hierarchical taxonomy terms select filter Views.

DESCRIPTION

The filter gets ability to use Ajax in select taxonomy terms
on exposed filter form.
It is work with another same filters and any other.

INSTALLATION

Install as usual, see http://drupal.org/node/70151 for further information.

USE

The module add a special filter to view names
"Content: Has a taxonomy term (with the hierarchy)".
Then necessary set the taxonomy vocabulary whose terms will be filtering
the nodes and optional label for added block of fileds. That's all.

RESTRICTIONS

The filter will not work if option 'autosubmit' is enabled
in 'Exposed form style' for view.


РУССКИЙ

ОПИСАНИЕ

Модуль добавляет к Views фильтр, который позволяет выбирать термины таксономии
с учетом иерархии. Фильтр использует jQuery и работает совместно с другими фильтрами,
в том числе и такими же, но для других словарей.
Полное описание: http://serho.ru/custmodules/filtr-dlya-views-s-ierarhicheskim-vyborom-terminov

УСТАНОВКА

Как обычно - распаковать модуль и включить (модуль находится в группе Views).
Подробности установки модулей в Друпал: http://drupal.org/node/70151

ИСПОЛЬЗОВАНИЕ

В настройках фильтров Views добавляется пункт "Content:Has a taxonomy term (with the hierarchy)".
Настройки достаточно очевидные, если нет, то подробная инструкция: http://serho.ru/custmodules/filtr-dlya-views-s-ierarhicheskim-vyborom-terminov/dobavlenie-filtra-dlya-views
Демонстрация работы: http://serho.ru/custmodules/filtr-dlya-views-s-ierarhicheskim-vyborom-terminov/demonstraciya-raboty-filtra-dlya

ОГРАНИЧЕНИЯ

Не будет работать при включенном autosubmit в настройках Exposed form style, даже не пытайтесь :).
Да и какой может быть автосабмит, если фильтр должен получить данные с сервера в этотм момент? :)
